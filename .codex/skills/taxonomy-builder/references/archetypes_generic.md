# Generic Archetypes

Use these archetypes when no explicit domain pack applies.
They are prompts for thinking, not mandatory buckets.

## Common chapter archetypes

- **Foundations / problem formulation**
  - Use when the field has competing task definitions, interfaces, or assumptions.

- **Core methods / components**
  - Use when the main reader question is how systems are built or what mechanism changes matter.

- **Learning / adaptation / coordination**
  - Use when systems improve over time, optimize policies, or coordinate multiple actors/components.

- **Evaluation / benchmarks**
  - Use when comparability, metrics, and protocol choices are major sources of disagreement.

- **Safety / robustness / governance**
  - Use when risk surfaces are substantive, not just a short caveat.

- **Applications / deployment settings**
  - Use only if application differences meaningfully reorganize the literature.

## Generic builder expectations

The compatibility fallback builder is heuristic.
Treat it as a baseline scaffold to refine, not as the final taxonomy if the domain is well understood.
