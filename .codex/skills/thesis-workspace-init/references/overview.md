# Overview

`thesis-workspace-init` 的目标不是“把论文写出来”，而是先把**工作区、材料区、中间层、交付层**理清楚。

它服务的是后续所有 graduate-paper skills，因此输出必须稳定、克制、可复用。

## 核心原则

1. 先把目录和材料放对，再谈内容重构。  
2. `codex_md/` 是思考层，`chapters/` 是交付层。  
3. 不直接改正文，只建立中间层骨架与材料索引。  
4. 必须显式写出当前还缺什么材料，而不是默认都齐了。  
5. 输出必须帮助后续 skill 快速接手，而不是写一篇说明文。  

## 最小初始化结果

至少应具备：

- `codex_md/material_index.md`
- `codex_md/material_readiness.md`
- `codex_md/missing_info.md`
- `codex_md/question_list.md`
- `codex_md/00_thesis_outline.md`
- `claude_md/review_checklist.md`

## 与通用 `workspace-init` 的关系

这个 skill 不是通用 workspace-init 的替代品，而是**毕业论文场景专用补充层**：

- 通用 `workspace-init` 负责通用 artifact-first 骨架
- `thesis-workspace-init` 负责毕业论文特有的中间层与材料盘点
