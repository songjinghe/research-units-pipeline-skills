# Introduction Jobs

## Core function

The Introduction should make the reader adopt the survey’s lens before the body starts.

## Recommended paragraph jobs

Use only the jobs that the evidence and scope justify.

1. Motivation
- Why the topic is difficult to compare now
- Why architecture names alone are not enough

2. Boundary
- What counts as in-scope
- Which neighboring topics are explicitly excluded or secondary

3. Lens commitment
- Which 2-4 lenses organize the survey
- Why those lenses are stable enough to compare across papers

4. Methodology note (exactly once)
- time window
- candidate pool size and/or core set size
- evidence mode when it materially affects interpretation
- phrased as survey methodology, not execution log

5. Reader contract
- how to read claims in the rest of the paper
- where comparisons are strong versus provisional

6. Light organization note
- optional and short
- should not dominate the introduction

## Methodology note rule

Put the methodology note once.
Do not spread “abstract-only / provisional / verification needed” boilerplate across all H3 sections.

## Introduction failure modes

Common problems:
- openers that narrate the section instead of making a claim
- overlong boundary paragraphs with no decision relevance
- a method note that reads like a run log
- a contributions paragraph that is only structure narration

## Good editing question

After paragraph 1, can a reader answer:
- what is hard to compare here?
- what lens this survey will use?
- why that lens is better than the obvious default?
