# Paragraph Jobs

Use paragraph jobs as coverage targets, not a fixed paragraph count.

## Core jobs for an H3 body

A strong subsection usually covers most of these jobs:
- opener / thesis
- mechanism or setup
- one or more explicit contrasts
- evaluation anchor
- cross-paper synthesis
- limitation or boundary condition
- local conclusion / handoff

## Minimal viable subsection

If the pack is thin, do not pad to a target count.
A minimal viable subsection should still do three things:
- state the tension and thesis
- give at least one concrete comparison
- name one limitation that changes interpretation

## Rich-pack subsection

If the pack is rich, distribute jobs across multiple paragraphs, but let the pack decide thickness.
Do not force all subsections to have the same length.

## Ordering rule

Preferred order is flexible.
Good orderings include:
- tension -> contrast -> evaluation -> limitation
- failure -> mechanism contrast -> evaluation -> synthesis
- protocol -> comparison -> boundary -> takeaway

## Writer reminder

Coverage matters more than count.
If a new paragraph does not add a new job or a sharper execution of a job, it probably should not exist.
