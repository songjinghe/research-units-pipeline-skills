# Overview

`question_list.md` 是这条毕业论文 Pipeline 的控制面。

它应该回答的不是“论文现在写到哪了”，而是：

- 这轮要解决什么问题？
- 为什么这些问题重要？
- 先修哪些，后修哪些？
- 修到什么程度算过关？

## 基本原则

1. 问题必须按优先级排序。  
2. 结构问题优先于文风问题。  
3. 每条问题都必须能落到具体文件或章节。  
4. 每条问题都要有可检查的验收口径。  

## 典型问题来源

- 章节角色失衡
- 与毕业论文主线不一致
- 同一术语多种说法
- 图表与正文脱节
- 引用与论断不匹配
- `latexmk` 编译 warning
- 章首 / 章末模板味过强
