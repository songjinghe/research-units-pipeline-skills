# Evaluation Anchor Rules

## Goal

Evaluation bullets should help later writing anchor claims to protocol, not add generic caveats.

## Prefer

- named benchmarks/datasets when available
- metric names or human-eval protocol when extractable
- explicit notes about shared vs non-shared protocol before cross-paper comparison
- constraint context such as budget, tool access, horizon, or threat model when present
- keep benchmark inventory as metadata for later filtering / anchoring, not as a ready-made prose sentence

## Avoid

- generic caution bullets that merely say evidence is sparse
- method-superiority language when benchmark/metric are not aligned
- invented baseline/model naming details
