# 毕业论文修改与沉淀 SOP

> 版本：v2  
> 适用场景：基于学校模板、既有论文、Overleaf 源稿、PDF、bib 与中间工作文档，重构并完成一篇可提交的毕业论文。  
> 核心目标：把“已有论文材料”加工成“主线统一、结构顺畅、证据完整、可编译交付”的毕业论文。  
> 方法定位：这不是线性写作流程，而是一个**问题驱动、Markdown 中间层主导、反复回环的论文工程流程**。

## 1. 这套流程的本质

这套流程不是“直接写论文”，也不是“把几篇 paper 拼起来”，而是：

1. **先还原材料，再围绕主线重构。**  
2. **先在 Markdown 中间层想清楚，再回写到 TeX 交付层。**  
3. **先解决结构与证据问题，再解决文风问题。**  
4. **先保证引用、数据、术语正确，再做润色与去 AI 味。**  
5. **真实执行不是瀑布流，而是多轮迭代闭环。**

一句话说，这套 SOP 更接近：

> `问题清单 → 语义定位 → Markdown 重构 → tex 回写 → 编译 → 复查 → 回到问题清单`

## 2. 目录分层与职责

建议长期固定以下目录分层：

- `main.tex`：论文总入口，负责装配封面、摘要、目录、正文、附录、致谢、成果与参考文献。  
- `chapters/`：最终交付版正文 `.tex`。  
- `abstract/`、`preface/`、`acknowledgement/`、`achievements/`：非正文但属于正式交付物的部分。  
- `references/`：参考文献库与样式文件。  
- `pdf/`：已发表或已投稿论文 PDF。  
- `Overleaf_ref/`：既有论文的源稿、修回稿、补充材料。  
- `codex_md/`：中间工作层，是**论文重构工作区**。  
- `claude_md/`：复查清单、终稿核对记录。  
- `tmp_layout/`、`tmp_layout2/`：图表试排与拼版预演。  

其中最关键的是：

- `codex_md/` 是**思考区 / 重构区**  
- `chapters/` 是**交付区 / 定稿区**

不要把二者混用。

### 2.1 `codex_md/` 内部再分工

`codex_md/` 不是普通草稿目录，而是问题驱动的中间工作层，建议固定包括：

- `00_thesis_outline.md`：总体主线、研究问题、章节角色、二级标题规划  
- `01...06...md`：各章 Markdown 中间稿  
- `question_list.md`：本轮问题清单、修改目标、agent 协作入口  
- `中文写作要求.md`：术语、句式、图表、指标、引用规范  
- `GPT口癖与高频用词调研.md`：AI 润色约束与去模板味参考  
- `mermaid/`：概念图、流程图、框架图草稿  
- 其他：图表计划、文件索引、补证据记录

## 3. 真实流程的骨架：主循环 + 支线模块

### 3.1 主循环

这是最贴近当前真实工作方式的主流程：

1. 从 `question_list` 明确本轮目标  
2. 用语义检索定位相关章节与上下文  
3. 在 Markdown 中间层重构内容  
4. 回写对应 `tex`  
5. 编译 `main.pdf`  
6. 用 `review_checklist` 记录问题  
7. 决定问题回到哪一层修：  
   - 回 `outline`  
   - 回 `question_list`  
   - 回某章 `.md`  
   - 还是直接在 `tex` 层收口  

所以真实执行不是 `0 → 1 → 2 → 3 → ...` 一次走完，而是会反复往返。

### 3.2 支线模块

以下模块不是最后才做，而是会在主循环中并行插入：

- 文献增强与引用校验  
- 图表规划与版面预演  
- 写作规范与术语统一  
- 非正文部分并行维护  
- 多 agent 交叉审稿与复查

## 4. 标准执行路径

下面这些阶段仍然有顺序，但要理解为“主循环中反复调用的模块”，不是只能做一次的流水线。

### 阶段 0：项目初始化

目标：先把论文变成一个可维护工程。

具体动作：

1. 建立模板仓库，并确认 `main.tex` 可以编译。  
2. 建好材料区、工作区、成文区、复查区。  
3. 归档学校模板、参考论文、PDF、Overleaf 源稿、bib。  
4. 在 `main.tex` 中确认基础参数：  
   - 年份  
   - 学号  
   - 中英文题目  
   - `draftfigure`  
   - `docstyle`  
   - `refstyle`

输出物：

- 可编译模板仓库  
- 初始目录结构  
- 初始 `main.pdf`

### 阶段 1：从 template 与历史材料还原初始 Markdown

这是起点，但不要把它理解成“只从 template 还原”。

具体流程包括：

1. **先根据 template 中的参考文献还原出对应的 Markdown 内容；**  
2. 从学校模板、已有 `tex`、Overleaf 源稿、PDF、参考文献中抽取能复用的内容；  
3. 把这些材料转成 Markdown 初稿，而不是直接在 `tex` 里改；  
4. 标记缺失信息：  
   - 待补实验细节  
   - 待补图注  
   - 待补引用  
   - 待补数字核验  

为什么这样做：

- 方便后续归并与重组；  
- 降低被排版打断的成本；  
- 先做“材料资产盘点”。

输出物：

- 初始 Markdown 章节材料  
- 待补信息清单  
- 材料索引

### 阶段 1.5：建立问题清单与本轮目标

这一步在当前真实流程里非常关键，不能省略。

建议固定使用：

- `codex_md/question_list.md`

它的作用不是备忘录，而是：

1. 记录本轮要解决的结构问题  
2. 明确章节重心与修改边界  
3. 记录哪些内容要重构、哪些只需润色  
4. 记录多 agent 协作方案  
5. 记录本轮的验收标准

一个标准的问题单至少应包括：

- 问题描述  
- 涉及章节  
- 为什么这是问题  
- 修改方向  
- 是否需要 agent 并行讨论  
- 最终验收口径

输出物：

- 本轮问题单  
- 优先级排序  
- 涉及章节与文件列表

### 阶段 2：把既有论文映射到“毕业论文角色”

这里不是简单“paper 对 chapter”的分配，而是毕业论文重构的起点。

具体流程包括：

1. **根据你指定文件中的论文，判断它们分别对应哪些章节，并生成相应的 `.md` 文件；**  
2. 对每篇论文建立映射：  
   - 论文  
   - 对应研究问题  
   - 对应章节  
   - 在毕业论文中的角色  
   - 关键实验  
   - 可复用图表  
   - 关键引用  
3. 判断这篇 paper 在毕业论文里是：  
   - 方法主体  
   - 背景支撑  
   - 下游验证载体  
   - 系统实现/部署证据  
   - 局限与展望素材

输出物：

- 按章拆分的 Markdown 草稿  
- 论文与章节角色映射表  
- 各章内容边界

### 阶段 2.5：从 paper 到毕业论文的重构

这一阶段要单独强调，因为它是当前流程里最容易被低估、但实际最核心的部分。

原则是：

> 不是翻译 paper，不是摘抄 paper，而是围绕毕业论文主线重构。

具体动作：

1. 重定章节目标：这一章要回答什么问题  
2. 重排内容比重：  
   - 哪些内容强化  
   - 哪些内容弱化  
   - 哪些内容上移到第 1/2 章统一讲  
3. 重写章节承接：  
   - 与上一章的关系  
   - 与下一章的关系  
4. 把“原论文式叙事”改成“毕业论文式叙事”：  
   - 更强调主线  
   - 更强调统一符号和口径  
   - 更强调系统性与局限  
   - 更强调任务级证据而非 paper 自身卖点

建议增加一张“重构准则表”，列为：

- 章节  
- 来源论文/材料  
- 本章角色  
- 保留内容  
- 弱化内容  
- 需新增内容  
- 与前后章关系  
- 当前状态

输出物：

- 重构后的章节定位  
- 重构准则表  
- 更贴合毕业论文主线的章节 Markdown

### 阶段 3：Markdown 对齐、归并与统一

这一阶段决定整篇论文能不能像“一篇论文”，而不是几篇论文的拼接。

具体流程包括：

1. **对这些 Markdown 内容进行对齐、整理和归并，逐步理顺整体结构与上下文逻辑；**  
2. 统一以下内容：  
   - 研究主线  
   - 研究问题  
   - 术语与缩写  
   - 符号与变量  
   - 指标与协议  
   - 图表口径  
3. 决定哪些内容：  
   - 放第 2 章统一解释  
   - 放第 3～5 章展开  
   - 放附录承接  
4. 按 `question_list` 中的问题回填并迭代，直到章节结构基本稳定。  

这一阶段的核心输出不是句子，而是：

- 定稿版 `outline`  
- 各章稳定的二级标题  
- 术语表 / 符号表 / 指标表  
- 证据缺口清单

### 阶段 3.5：图表规划与版面预演

这一步必须显式写进 SOP，因为当前真实流程明显不是“文写完再补图”。

建议固定使用：

- `codex_md/mermaid/`：概念图与流程图草稿  
- `tmp_layout/`、`tmp_layout2/`：图表拼版与版面预演

具体流程包括：

1. 先确定本章需要哪些图表来支撑主线  
2. 用 `mermaid` 或草图先画概念图 / 流程图  
3. 再做临时拼版，检查：  
   - 图表是否过密  
   - 图文节奏是否合理  
   - 哪张图应该提前或后移  
4. 最后再决定图表落位与图注口径

输出物：

- 图表计划表  
- 图表草稿  
- 图文映射关系  
- 临时版式预演结果

### 阶段 4：回写 `tex` 并进入交付层

当 Markdown 层基本稳定后，再回写到 `chapters/*.tex`。

具体流程包括：

1. **在此基础上继续生成 outline 和 tex 内容，并处理章节之间承上启下的衔接与对齐；**  
2. 把 Markdown 中已经理顺的结构和内容回写到 `chapters/*.tex`；  
3. 同步处理：  
   - 图表插入与编号  
   - 公式与变量解释  
   - 引用插入  
   - 交叉引用  
   - 章首导言与章末小结  
4. 特别注意章节衔接：  
   - 第 2 章承接第 1 章  
   - 第 3～5 章体现递进关系  
   - 第 6 章统一回扣前三章

关键原则：

- 不在 `tex` 中重新发明结构  
- `tex` 负责交付，不负责重新思考  
- 结构性问题优先回 Markdown 层解决

输出物：

- 可编译的章节 `.tex`  
- 初版完整 `main.pdf`

### 阶段 4.5：非正文部分并行完善

这一步不能拖到最后随手填。

需要并行维护的非正文部分包括：

- `abstract/abstract.tex`  
- `abstract/abstract-en.tex`  
- `preface/inner-cover.tex`  
- `preface/inner-cover-en.tex`  
- `chapters/appendix.tex`  
- `acknowledgement/`  
- `achievements/`  
- `preface/member-list.tex`

核查重点：

1. 中英文题目一致  
2. 中英文摘要数字与口径一致  
3. 附录只放补充性内容，不重复正文主线  
4. 封面与模板字段完整  
5. 致谢、成果、名单等模板项不遗漏

输出物：

- 非正文部分定稿  
- 与正文口径一致的摘要、附录、封面与模板项

### 阶段 5：文献增强与多轮校验

这是当前流程里新增且值得固定进 SOP 的模块。

此外，本轮还补充了文献增强相关能力，例如通过 `grok-mcp` 扩充参考文献来源，并结合 GPT 对文献进行多轮校验，以提升引用的完整性与准确性。

建议固定流程如下：

1. 先找出必须有引用支撑的句子：  
   - 事实性描述  
   - 方法分类  
   - 经典结论  
   - 指标定义  
   - 数据来源  
2. 用检索工具扩展候选文献，优先补：  
   - 综述  
   - 奠基性工作  
   - 最近两年的代表性工作  
   - 与本章直接相关的对照工作  
3. 用 GPT 或其他模型做多轮校验，但只做以下事：  
   - 是否遗漏关键工作  
   - 引用是否匹配论断  
   - 年份、作者、venue 是否一致  
   - 是否存在误引  
4. 人工确认后再写入 `references/*.bib`，再回写正文。

输出物：

- 增强后的文献库  
- 每章引用补强结果  
- 引用核验记录

### 阶段 6：编译、排版与终稿复查

这一阶段不只是“能编译通过”，而是一个质量闭环。

#### 6.1 编译通过

目标是生成稳定的 `main.pdf`。

建议使用：

- `latexmk -xelatex`

#### 6.2 warning 分级处理

把 warning 分成三类：

- **P1 阻断提交**：编译失败、引用丢失、关键文件缺失  
- **P2 必须修复**：关键图表未引用、公式未解释、数字不一致、严重排版问题  
- **P3 可记录观察**：轻微 `overfull` / `underfull`、个别模板 warning、局部风格问题

#### 6.3 数据与证据核验

当前真实流程里，这一步非常数据化，建议显式写入：

1. 主表逐项比对原论文  
2. 消融表逐项比对  
3. 中英文摘要数字一致性  
4. cite key 是否全部有效  
5. 术语、缩写、符号是否统一  
6. 自引比例是否合理

#### 6.4 模板交付模式检查

提交前检查：

- `draftfigure` 是否正确  
- `docstyle` 是否正确  
- `refstyle` 是否正确  
- 盲审版 / 查重版 / 终稿版是否切换正确  

#### 6.5 问题回写闭环

所有问题都要回写到：

- `claude_md/review_checklist.md`  
- 必要时回写到 `codex_md/question_list.md`

然后再决定：

- 回 `outline` 修  
- 回 `.md` 修  
- 回 `tex` 修  
- 还是作为可接受 warning 记录保留

输出物：

- 稳定可编译 PDF  
- 复查清单  
- 已关闭问题记录

### 阶段 6.6：多 agent 交叉审稿与复查

当前真实流程里已经出现多 agent 交叉思考与交叉复核，因此建议正式写进 SOP。

推荐模式：

1. Agent A：初步修改  
2. Agent B：复查 A 的结果并指出问题  
3. Agent C：复核 B 对 A 的评价是否准确  
4. 主控综合 A/B/C 结果做最终调整  
5. 以 B 与 C 基本达成一致作为收口条件

适用场景：

- 章节重构  
- 结构争议  
- 文风争议  
- 复查阶段的疑难问题

### 阶段 7：AI 润色与去 AI 味

AI 可以在中前期参与结构审查、方案讨论和定位，但**最终润色必须放在结构、引用、数据稳定之后**。

具体流程包括：

1. **最后再交给 AI 做进一步润色，同时尽量弱化明显的 AI 口癖，让表达更自然。**  
2. 润色重点不是“写得更像 AI”，而是：  
   - 更像作者本人  
   - 更像学位论文  
   - 更少模板腔与宣传腔  
3. 优先处理：  
   - 章首导言  
   - 章末小结  
   - 贡献描述  
   - 总结与展望  
   - 空泛过渡句  
4. 稀释这些高风险模式：  
   - 过于固定的“首先/其次/然后/最后”  
   - “值得注意的是/需要强调的是/综上所述”高频重复  
   - 英文里的 `delve into`、`At its core`、`plays a pivotal role` 等模板句  
   - 中文里的“赋能/闭环/抓手/落地/口径”等非学术表达

建议的润色顺序：

1. 先改结构性重复  
2. 再改句式模板化  
3. 最后改局部措辞和节奏

输出物：

- 更自然的终稿  
- 去模板化后的导言、小结、总结与展望

## 5. 真实执行中的四个闭环

为了让 SOP 更贴近真实流程，建议把以下四个闭环写死在方法论里。

### 5.1 结构闭环

`outline → 某章 md → 发现章节失衡 → 回 outline / question_list`

### 5.2 内容闭环

`paper / Overleaf / PDF → 抽取内容 → 重构叙事 → 发现仍像原论文 → 再重构`

### 5.3 排版闭环

`tex → 编译 → 发现 warning / 图表位置 / 交叉引用问题 → 回 tex 或图表规划`

### 5.4 文风闭环

`正文稳定 → AI 润色 → 发现模板腔 / AI 味 → 回写作规范与措辞控制`

## 6. 工具分工建议

为了让 SOP 可复用，建议把工具职责固定下来：

- **语义定位 / 不确定位置**：优先用 `fast_context_search`  
- **精确关键词定位**：用 `rg` / grep  
- **查看已有内容**：Read / `sed` / `nl`  
- **修改 Markdown / TeX**：Edit / patch  
- **补文献与外部事实**：`grok-mcp` 或其他可追溯检索工具  
- **全局编译验证**：`latexmk -xelatex`  
- **结构讨论 / 交叉复查**：多个 agent 并行  
- **AI 润色**：放在结构稳定、引用稳定之后

一条重要经验是：

> 先用语义搜索找到上下文，再做局部修改，远比直接在多个章节里盲改高效。

## 7. 每一轮修改的标准循环

建议把单轮工作固定成以下 cycle：

1. 明确本轮目标：补结构、补证据、补图、补引用、修排版，还是润色  
2. 在 `question_list` 中写清问题与验收标准  
3. 用语义检索定位相关上下文  
4. 先改 Markdown 中间层  
5. 必要时同步改图表规划与版面预演  
6. 再回写 `tex`  
7. 编译并查看 PDF  
8. 把问题写回 `review_checklist`  
9. 决定是否回到 `outline` / `question_list` / `.md`  
10. 结构稳定后再进入 AI 润色

## 8. 常见风险与规避策略

- **风险 1：直接在 TeX 里边改边想。**  
  规避：先在 Markdown 中间层完成结构重组。  

- **风险 2：把多篇论文直接拼成毕业论文。**  
  规避：先做“章节角色定义 + 重构准则表”，再写正文。  

- **风险 3：把重构误当直译。**  
  规避：明确哪些 paper 内容要保留、弱化、上移或重写。  

- **风险 4：先润色，后补引用。**  
  规避：文献完整性必须先于文风优化。  

- **风险 5：AI 直接生成参考文献或实验事实。**  
  规避：AI 只辅助检索与核验，最终条目必须人工确认。  

- **风险 6：图表拖到最后才处理。**  
  规避：把图表规划与版面预演前置。  

- **风险 7：非正文部分最后才想起来。**  
  规避：摘要、封面、附录、成果、致谢并行维护。  

- **风险 8：章节各写各的，术语口径漂移。**  
  规避：单独维护术语、指标、符号与写作规范文档。  

- **风险 9：问题没有沉淀成下一轮输入。**  
  规避：所有关键问题必须写回 `question_list` 或 `review_checklist`。  

## 9. 可复用资产清单

未来每做一篇毕业论文，都建议沉淀以下资产：

- 一份 `SOP`  
- 一份 `thesis_outline` 模板  
- 一份 `question_list` 模板  
- 每章 Markdown 模板  
- 图表规划模板  
- 术语统一表  
- 引用核验清单  
- 终稿复查清单  
- AI 去模板味词表与替换建议  
- 章节重构与交付跟踪表

## 10. 一句话版本

这套 SOP 的一句话总结是：

> 先从模板、既有论文和 Overleaf 源稿中还原材料，再通过 `question_list` 驱动 Markdown 中间层重构；随后把内容、图表、引用与非正文部分同步回写到 TeX，并在“编译—复查—回填问题单”的迭代闭环中逐步收口；最后再通过 AI 做有限润色和去模板味，把论文从“材料集合”打磨成“结构统一、证据完整、表达自然的终稿”。
