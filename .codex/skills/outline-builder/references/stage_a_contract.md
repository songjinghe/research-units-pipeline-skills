# Stage A Contract

Every H3 subsection must stay bullets-only and auditable.

## Required fields

Each subsection should retain these fields in this order:
- `Intent:`
- `RQ:`
- `Scope cues:` when a usable hint exists
- `Evidence needs:`
- `Expected cites:`
- `Concrete comparisons:`
- `Evaluation anchors:`
- `Comparison axes:`

## Quality bar

The bullets should be:
- specific enough for later paper mapping
- phrased as coverage requirements, not prose narration
- conservative when evidence is still thin
- stable across domains by using assetized defaults and routing rules

## Avoid

- narration stems like `This subsection discusses the topic.`
- TODO/TBD placeholders
- repeated generic bullets with no topic terms
- bullets that cannot be mapped to papers or evidence later
